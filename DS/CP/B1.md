Tuyệt vời! Biểu đồ UML bạn cung cấp là một ví dụ vô cùng trực quan và kinh điển về cách áp dụng mẫu **Decorator** 🎁 để tính toán giá trị biểu thức toán học.

Trong bài toán này, thay vì tạo ra các lớp biểu thức phức tạp, chúng ta sẽ bắt đầu với một con số cơ bản (`BieuThucDonGian`), sau đó "bọc" (decorate) nó dần dần bằng các phép toán `Cong`, `Tru`, `Nhan`, `Chia`. Mỗi lần bọc, chúng ta lại có một đối tượng biểu thức mới với giá trị được cập nhật.

Dưới đây là mã nguồn Java hoàn chỉnh bám sát theo biểu đồ UML của bạn:

### 1. Cấu trúc các lớp (Classes)

```java
// Lớp trừu tượng gốc (Component)
abstract class BieuThuc {
    public abstract float giaTri();
    public abstract String bieuThuc();
}

// Đối tượng cơ bản cần được trang trí (Concrete Component)
class BieuThucDonGian extends BieuThuc {
    private float toanHang;

    public BieuThucDonGian(float toanHang) {
        this.toanHang = toanHang;
    }

    @Override
    public float giaTri() {
        return toanHang;
    }

    @Override
    public String bieuThuc() {
        return String.valueOf(toanHang);
    }
}

// Lớp Decorator cơ sở (Base Decorator)
abstract class BieuThucDecorator extends BieuThuc {
    protected BieuThuc bieuThuc; // Biến này giữ đối tượng bị "bọc" ở bên trong

    public BieuThucDecorator(BieuThuc bieuThuc) {
        this.bieuThuc = bieuThuc;
    }

    @Override
    public float giaTri() {
        return bieuThuc.giaTri();
    }

    @Override
    public String bieuThuc() {
        return bieuThuc.bieuThuc();
    }
}

// --- CÁC LỚP DECORATOR CỤ THỂ (Concrete Decorators) ---
// Lưu ý: Tôi dùng kiểu 'float' cho toán hạng ở tất cả các lớp thay vì 'int' 
// ở Nhan/Chia như biểu đồ để đảm bảo tính đồng nhất khi tính toán.

class Cong extends BieuThucDecorator {
    private float toanHang;

    public Cong(BieuThuc bieuThuc, float toanHang) {
        super(bieuThuc);
        this.toanHang = toanHang;
    }

    @Override
    public float giaTri() {
        return bieuThuc.giaTri() + toanHang;
    }

    @Override
    public String bieuThuc() {
        return "(" + bieuThuc.bieuThuc() + " + " + toanHang + ")";
    }
}

class Tru extends BieuThucDecorator {
    private float toanHang;

    public Tru(BieuThuc bieuThuc, float toanHang) {
        super(bieuThuc);
        this.toanHang = toanHang;
    }

    @Override
    public float giaTri() {
        return bieuThuc.giaTri() - toanHang;
    }

    @Override
    public String bieuThuc() {
        return "(" + bieuThuc.bieuThuc() + " - " + toanHang + ")";
    }
}

class Nhan extends BieuThucDecorator {
    private float toanHang;

    public Nhan(BieuThuc bieuThuc, float toanHang) {
        super(bieuThuc);
        this.toanHang = toanHang;
    }

    @Override
    public float giaTri() {
        return bieuThuc.giaTri() * toanHang;
    }

    @Override
    public String bieuThuc() {
        return "(" + bieuThuc.bieuThuc() + " * " + toanHang + ")";
    }
}

class Chia extends BieuThucDecorator {
    private float toanHang;

    public Chia(BieuThuc bieuThuc, float toanHang) {
        super(bieuThuc);
        this.toanHang = toanHang;
    }

    @Override
    public float giaTri() {
        if (toanHang == 0) {
            throw new ArithmeticException("Không thể chia cho 0");
        }
        return bieuThuc.giaTri() / toanHang;
    }

    @Override
    public String bieuThuc() {
        return "(" + bieuThuc.bieuThuc() + " / " + toanHang + ")";
    }
}

```

### 2. Áp dụng tính giá trị biểu thức (Chương trình chính)

Bây giờ chúng ta sẽ dùng các lớp trên để tính 2 biểu thức theo yêu cầu trong đề bài:

1. $(6+9)*8 + 5$
2. $(8+6)/2*5 + 8-4$

```java
public class Main {
    public static void main(String[] args) {
        // Biểu thức 1: (6 + 9) * 8 + 5
        BieuThuc b1 = new BieuThucDonGian(6); // Bắt đầu với số 6
        b1 = new Cong(b1, 9);                 // Bọc phép cộng 9  -> (6 + 9)
        b1 = new Nhan(b1, 8);                 // Bọc phép nhân 8  -> (6 + 9) * 8
        b1 = new Cong(b1, 5);                 // Bọc phép cộng 5  -> (6 + 9) * 8 + 5
        
        System.out.println("Biểu thức 1: " + b1.bieuThuc());
        System.out.println("Kết quả 1  : " + b1.giaTri());
        System.out.println("----------------------------------");

        // Biểu thức 2: (8 + 6) / 2 * 5 + 8 - 4
        // Tính từ trái qua phải theo trật tự ngoặc và toán tử:
        BieuThuc b2 = new BieuThucDonGian(8); // Bắt đầu với số 8
        b2 = new Cong(b2, 6);                 // + 6
        b2 = new Chia(b2, 2);                 // / 2
        b2 = new Nhan(b2, 5);                 // * 5
        b2 = new Cong(b2, 8);                 // + 8
        b2 = new Tru(b2, 4);                  // - 4
        
        System.out.println("Biểu thức 2: " + b2.bieuThuc());
        System.out.println("Kết quả 2  : " + b2.giaTri());
    }
}

```

**Kết quả khi chạy chương trình:**

```text
Biểu thức 1: (((6.0 + 9.0) * 8.0) + 5.0)
Kết quả 1  : 125.0
----------------------------------
Biểu thức 2: (((((8.0 + 6.0) / 2.0) * 5.0) + 8.0) - 4.0)
Kết quả 2  : 39.0

```

Bạn có thể thấy sức mạnh của Decorator ở đây: chúng ta liên tục gán `b1 = new ... (b1, ...)` giống như việc lồng những con búp bê Nga vào nhau. Mỗi đối tượng mới đều bao bọc đối tượng cũ, tính toán kết quả của đối tượng cũ trước rồi mới thực hiện phép toán của riêng nó.
